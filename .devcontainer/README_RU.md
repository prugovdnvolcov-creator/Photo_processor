# Photo Agent Dev Container

Этот проект настроен для разработки в изолированной среде с помощью **Dev Containers** в VS Code. Это позволяет запускать и разрабатывать проект без установки Python, зависимостей или других инструментов на ваше локальное устройство.

## 📋 Требования

1. **Visual Studio Code** - [Скачать](https://code.visualstudio.com/)
2. **Docker Desktop** (или Docker Engine) - [Скачать](https://www.docker.com/products/docker-desktop/)
3. **Расширение "Dev Containers"** для VS Code (устанавливается автоматически при открытии проекта)

## 🚀 Инструкция по запуску

### Шаг 1: Откройте проект в VS Code
```bash
code /workspace
```

### Шаг 2: Откройте проект в контейнере
1. Нажмите `F1` или `Ctrl+Shift+P` (на Mac: `Cmd+Shift+P`)
2. Введите и выберите: **`Dev Containers: Reopen in Container`**
3. VS Code начнет сборку контейнера (первый запуск может занять 2-5 минут)
4. После завершения VS Code автоматически откроется внутри контейнера

### Шаг 3: Дождитесь установки зависимостей
После открытия контейнера автоматически выполнится команда:
```bash
cd photo_agent && pip install -e .[dev]
```
Это установит все необходимые зависимости проекта.

### Шаг 4: Создайте директории для входных/выходных файлов
Выполните задачу VS Code:
1. Нажмите `Ctrl+Shift+P` (на Mac: `Cmd+Shift+P`)
2. Выберите: **`Tasks: Run Task`**
3. Выберите: **`Create Input/Output Directories`**

Или выполните в терминале:
```bash
mkdir -p photo_agent/input photo_agent/output
```

## 🎯 Как использовать

### Запуск обработки изображений

#### Вариант 1: Через панель отладки (рекомендуется)
1. Перейдите на вкладку **"Run and Debug"** (значок треугольника слева)
2. Выберите конфигурацию **"Python: Photo Agent CLI"**
3. Поместите изображения в папку `photo_agent/input`
4. Нажмите зеленую кнопку запуска (F5)
5. Обработанные изображения появятся в `photo_agent/output`

#### Вариант 2: Через задачи VS Code
1. Нажмите `Ctrl+Shift+P` (на Mac: `Cmd+Shift+P`)
2. Выберите: **`Tasks: Run Task`**
3. Выберите: **`Run Photo Agent CLI`**

#### Вариант 3: Через терминал
```bash
cd photo_agent
python -m photo_agent.src.cli --input ./input --output ./output
```

### Запуск тестов
1. Нажмите `Ctrl+Shift+P`
2. Выберите: **`Tasks: Run Task`**
3. Выберите: **`Run Tests`**

Или через отладку:
1. Выберите конфигурацию **"Python: Run Tests"**
2. Нажмите F5

### Форматирование и линтинг кода
- **Форматирование**: `Tasks: Run Task` → `Format Code (Black)`
- **Линтинг**: `Tasks: Run Task` → `Lint Code (Flake8)`

## 🛠️ Доступные конфигурации отладки

| Название | Описание |
|----------|----------|
| **Python: Photo Agent CLI** | Запуск основного CLI приложения с аргументами |
| **Python: Current File** | Запуск текущего открытого файла Python |
| **Python: Run Tests** | Запуск тестов с покрытием кода |

## 📁 Структура проекта в контейнере

```
/workspace/
├── .devcontainer/          # Конфигурация контейнера
├── .vscode/               # Настройки VS Code
├── photo_agent/           # Исходный код проекта
│   ├── src/              # Основной код
│   ├── tests/            # Тесты
│   ├── input/            # Входные изображения (создать вручную)
│   └── output/           # Выходные изображения (создать вручную)
└── photo_processor       # Дополнительный скрипт
```

## 🔧 Полезные команды

| Команда | Описание |
|---------|----------|
| `Dev Containers: Reopen in Container` | Открыть проект в контейнере |
| `Dev Containers: Close Remote Connection` | Закрыть соединение с контейнером |
| `Dev Containers: Clean Up Dev Containers...` | Удалить старые контейнеры |
| `Tasks: Run Task` | Выполнить предзаданную задачу |
| `Terminal: Run Active File` | Запустить текущий файл в терминале |

## ❓ Решение проблем

### Контейнер не запускается
1. Убедитесь, что Docker запущен
2. Проверьте, что у вас есть права на запуск Docker
3. Попробуйте перезапустить Docker Desktop

### Зависимости не устанавливаются
1. Откройте терминал в контейнере (`Ctrl+``)
2. Выполните вручную: `cd photo_agent && pip install -e .[dev]`

### Ошибки с доступом к файлам
Убедитесь, что папки `input` и `output` существуют:
```bash
mkdir -p photo_agent/input photo_agent/output
```

## 📝 Примечания

- Все изменения кода сохраняются на вашем хосте (в папке `/workspace`)
- Контейнер изолирован от вашей основной системы
- При повторном открытии проекта контейнер запускается быстро (кэшируется)
- Для обновления зависимостей пересоберите контейнер: `Dev Containers: Rebuild Container`
